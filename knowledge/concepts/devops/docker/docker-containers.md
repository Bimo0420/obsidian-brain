# Docker Containers

**Docker Container** (контейнер) — это запущенный экземпляр Docker-образа. Это изолированная среда выполнения процесса, которая включает в себя код, среду выполнения, системные инструменты, библиотеки и настройки.

## Управление контейнерами

| Команда | Описание |
| --- | --- |
| `docker run [flags] image` | Создание и запуск нового контейнера. |
| `docker ps` | Список запущенных контейнеров. |
| `docker ps -a` | Список всех контейнеров (включая остановленные). |
| `docker stop <id/name>` | Грациозная остановка контейнера. |
| `docker kill <id/name>` | Принудительная остановка (SIGKILL). |
| `docker rm <id/name>` | Удаление остановленного контейнера. |
| `docker start <id/name>` | Запуск ранее остановленного контейнера. |
| `docker restart <id/name>` | Перезапуск контейнера. |

## Взаимодействие и отладка

- `docker exec -it <id> bash` — выполнение команды (например, вход в shell) внутри запущенного контейнера.
- `docker logs -f <id>` — просмотр логов контейнера в реальном времени.
- `docker stats` — мониторинг потребления ресурсов (CPU, RAM, Сеть) в реальном времени.
- `docker cp` — копирование файлов между хостом и контейнером.
- `docker inspect` — полная конфигурация контейнера (IP, Volumes, Env).

## Флаги запуска (`docker run`)

- `-d`: Фоновый режим (detached).
- `-p host:container`: Проброс портов.
- `-v host:container`: Монтирование томов или директорий.
- `-e KEY=VAL`: Переменные окружения.
- `--name`: Присвоение имени контейнеру.
- `--network`: Подключение к конкретной сети Docker.
- `--gpus all`: Проброс GPU (требует NVIDIA Container Toolkit).
- `--restart`: Политика перезапуска (например, `unless-stopped`).
