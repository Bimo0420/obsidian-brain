# Docker Networks

**Docker Networks** позволяют контейнерам взаимодействовать друг с другом и с внешним миром. Docker предоставляет драйверы для различных типов сетевого взаимодействия, обеспечивая изоляцию и обнаружение сервисов.

## Ключевые команды

| Команда | Описание |
| --- | --- |
| `docker network create name` | Создание новой сети. |
| `docker network ls` | Список всех сетей. |
| `docker network inspect name` | Подробная информация о сети и подключенных контейнерах. |
| `docker network connect net cont` | Подключение запущенного контейнера к сети. |
| `docker network disconnect net cont` | Отключение контейнера от сети. |
| `docker network rm name` | Удаление сети. |

## Основные драйверы сетей

- **bridge:** Драйвер по умолчанию. Контейнеры в одной bridge-сети могут общаться по IP или именам (если сеть пользовательская).
- **host:** Контейнер использует сетевой стек хоста напрямую (нет изоляции портов).
- **none:** Полная сетевая изоляция контейнера.
- **overlay:** Позволяет контейнерам на разных хостах (в Docker Swarm) общаться друг с другом.
