# Docker Volumes

**Docker Volumes** (тома) — это основной механизм для персистентного хранения данных, генерируемых и используемых Docker-контейнерами. В отличие от bind mounts, тома полностью управляются Docker и изолированы от структуры файловой системы хоста.

## Ключевые команды

| Команда | Описание |
| --- | --- |
| `docker volume create name` | Создание именованного тома. |
| `docker volume ls` | Список всех томов. |
| `docker volume inspect name` | Подробная информация (включая путь на хосте). |
| `docker volume rm name` | Удаление тома. |
| `docker volume prune` | Удаление всех неиспользуемых томов. |

## Использование

Тома подключаются к контейнеру с помощью флага `-v` или `--mount` при запуске:
`docker run -v myvolume:/app/data myimage`

## Типы хранилищ в Docker

1. **Volumes:** Хранятся в части файловой системы хоста, которая управляется Docker (`/var/lib/docker/volumes/` в Linux). Лучший способ обеспечить персистентность.
2. **Bind Mounts:** Привязка любой директории или файла на хосте к контейнеру. Зависят от структуры папок хоста.
3. **tmpfs Mounts:** Хранение данных только в памяти хоста (не записываются на диск).
