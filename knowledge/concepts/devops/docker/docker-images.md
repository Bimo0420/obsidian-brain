# Docker Images

**Docker Image** (образ) — это неизменяемый шаблон с инструкциями для создания Docker-контейнера. Он представляет собой упакованное приложение со всеми его зависимостями, системными библиотеками и настройками, организованное в виде набора доступных только для чтения слоев.

## Ключевые команды

| Команда | Описание |
| --- | --- |
| `docker build -t name:tag .` | Сборка образа из Dockerfile в текущей директории. |
| `docker pull image:tag` | Загрузка образа из удаленного реестра (например, Docker Hub). |
| `docker push image:tag` | Отправка образа в удаленный реестр. |
| `docker images` | Список всех образов на локальной машине. |
| `docker rmi image:tag` | Удаление образа. |
| `docker history image:tag` | Просмотр истории слоев образа. |
| `docker inspect image:tag` | Получение подробной информации об образе в формате JSON. |
| `docker image prune -a` | Удаление всех неиспользуемых образов. |

## Особенности

- **Слои (Layers):** Каждый образ состоит из серии слоев. При изменении Dockerfile пересобираются только измененные слои и те, что идут после них.
- **Неизменяемость (Immutability):** После сборки образ не меняется. Любые изменения состояния происходят только в контейнере.
- **Реестры (Registries):** Образы хранятся и распространяются через Docker Hub или приватные реестры (GCR, ECR, и др.).
