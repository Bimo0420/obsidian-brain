# GitOps

GitOps — методология управления инфраструктурой и деплоем приложений, при которой Git является единственным источником истины (Source of Truth) для конфигурации кластера.

## Принципы

- Конфигурация Kubernetes хранится в Git-репозитории.
- Изменения в репозитории автоматически синхронизируются с кластером через операторы (ArgoCD, Flux).
- Обеспечивается высокая прозрачность, аудит и возможность быстрого отката (rollback).

## GitOps в MLOps

Применение GitOps для деплоя моделей позволяет:
- Автоматизировать переобучение и обновление моделей (Continuous Deployment).
- Сократить циклы доставки изменений (Time-to-Market).
- Использовать декларативный подход к описанию ресурсов GPU и других инфраструктурных компонентов.

## Инструменты

- **ArgoCD**
- **Flux**
