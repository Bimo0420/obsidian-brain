# Containerization

Контейнеризация — фундаментальная практика современного MLOps, которая решает проблему воспроизводимости: модель работает одинаково в любых окружениях (ноутбук, CI/CD, продакшн). Контейнер упаковывает приложение вместе с зависимостями, конфигурацией и системными библиотеками в изолированный исполняемый образ.

## Ключевые преимущества в ML

- **Воспроизводимость:** Гарантия идентичного окружения (Python, CUDA, библиотеки) на любой машине.
- **Изоляция:** Предотвращение конфликтов зависимостей (например, разных версий torch или numpy).
- **Портируемость:** Образ работает локально, в облаках (GCP, AWS) и на bare-metal.
- **Версионирование:** Образ жестко привязан к состоянию кода и зависимостей.

## Лучшие практики (Docker)

- **Multi-stage builds:** Разделение тяжелого образа для обучения и легкого для инференса.
- **Volumes:** Не хранить веса моделей внутри образа; монтировать внешние тома.
- **Фиксация тегов:** Использовать конкретные версии (например, `v1.2.3`), избегать `latest`.
- **Resource Limits:** Всегда ограничивать CPU/GPU/RAM для предотвращения нехватки ресурсов на ноде.

## Специфика GPU

Для работы с GPU в контейнерах требуются:
- Базовые образы с поддержкой CUDA (например, `nvidia/cuda`).
- **NVIDIA Container Toolkit** на хост-машине.
- Специализированные серверы инференса (Triton, TorchServe).

## Антипаттерны

- Хранение секретов (API-ключи) внутри образа.
- Запуск от имени `root`.
- Хранение тяжелых весов моделей (гигабайты) внутри Docker-образа.
